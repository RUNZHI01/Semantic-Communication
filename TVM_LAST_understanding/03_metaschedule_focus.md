# MetaSchedule 聚焦：关键点、风险、可优化位

## 1. 当前关键点（必须先稳住）

1. 固定实验基线：`target`、线程策略、shape 桶、测量参数必须锁定。  
2. 热点优先：预算只给前 `3-8` 个耗时任务。  
3. 双预算模式：白天 quick、夜间 full，避免一次跑到底。  
4. 调优资产沉淀：所有轮次必须保留 tuning DB 与 trace。  
5. 单变量迭代：每天只改一类参数，保证可归因。

## 2. 风险清单（按影响优先）

| 风险 | 影响 | 监控信号 | 今晚动作 |
|---|---|---|---|
| 非热点任务耗尽预算 | 低收益高耗时 | 总时长上升但端到端几乎无提升 | 只保留热点 task 列表 |
| 测量噪声过大 | 误判“伪提升” | 同配置多次差异明显 | 固定负载 + 增加复测中位数 |
| 交叉编译配置错误 | 任务直接失败 | `Exec format error` 或运行崩溃 | 复核 target/triple/toolchain |
| shape 漂移 | DB 复用失效 | 同模型复跑命中率低 | 按桶固定 shape 与命名 |
| 日志不完整 | 无法复现 | 仅有“提升了”但无参数记录 | 强制使用模板记录关键字段 |

## 3. 今晚最值得投入的优化位

1. 搜索空间约束：给 ARMv8 场景收紧 tile/vectorize 范围，减少无效候选。  
2. Runner 稳定性：保证真机测量环境稳定，先保真再提速。  
3. 数据库 warm-start：同 shape 桶优先加载已有 DB。  
4. 结果筛选标准：把“提升幅度 + 方差”设为双指标，不只看单次最小值。  
5. 失败样本归档：对失败任务记录最小复现信息，便于次日快速回放。

## 4. 不建议今晚做的事

1. 同时改多个变量（无法归因）。  
2. 为长尾 shape 做深度调优（ROI 低）。  
3. 全量任务 full 跑满整夜（时间消耗失控）。  
4. 在没有 baseline 的情况下宣布“优化成功”。

## 5. 今晚完成线（Done Definition）

- 至少 1 条 quick 结果可复现，含 `baseline -> current` 对比。  
- full 任务已排程且仅覆盖热点。  
- 关键参数、日志路径、DB 路径在报告中可追溯。  
- 明早可以直接据日志继续下一轮，不需要重新摸索。
