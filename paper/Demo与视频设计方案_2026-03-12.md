## 1. 答辩定位与 Demo 目标

本文件今晚上台优先服务答辩叙事，而不是继续把 Demo 设计成 generic benchmark UI。推荐口径应围绕以下四点展开：

1. **先讲任务，不先讲框架。** 作品首先是一个面向弱网场景的语义视觉回传系统，上位机负责语义编码与弱网传输，飞腾侧负责视觉重建回传。
2. **先讲飞腾多核系统，再讲单点优化。** 数据面由飞腾 Linux 侧 current live 重建链承担，控制面由 OpenAMP/RTOS 承担，两者分离后系统边界更清楚，也更符合飞腾杯系统作品口径。
3. **今晚上台主路径是 `current live + OpenAMP control plane`。** 现场重点不是把 baseline 跑一遍，而是证明弱网语义回传真的跑起来了，并且控制面不是 PPT 概念。
4. **所有数字必须带 operating mode。** `4-core Linux performance mode` 负责 headline performance；`3-core Linux + RTOS demo mode` 负责台上 live 与安全控制。两种 mode 不能混写。

评审在观看今夜版 Demo 时，最容易被说服的不是“又一个 TVM/MNN 图表”，而是下面四件事：

1. **它真的是飞腾板上的 live 系统。** 需要看到板卡在线、artifact/runner 身份、remoteproc 或控制面状态，而不是只看静态截图。
2. **它解决的是弱网语义视觉回传问题。** 编码、弱网扰动、传输、飞腾侧重建这条数据面必须被看见。
3. **它是飞腾多核系统，不是单点脚本。** 界面上要显式出现数据面 / 控制面分离，以及 Linux 多核性能模式和 Linux+RTOS 演示模式的区别。
4. **它知道自己的边界。** 需要主动说明 OpenAMP 当前卖点是 control plane、admission、heartbeat、safe stop、fault handling，而不是“让推理更快”。

建议开场 20 秒直接使用下面这句：

> 我们今晚展示的不是 generic TVM benchmark，而是一套部署在飞腾平台上的弱网语义视觉回传系统：`4-core Linux performance mode` 负责多核高性能重建，`3-core Linux + RTOS demo mode` 负责 OpenAMP 安全控制面，台上重点展示当前 live 重建链和真实 control plane 闭环。

## 3. 推荐演示结构

### 3.1 推荐上台顺序（四步）

| 步骤 | 台上动作 | 要讲清楚什么 | 推荐时长 |
|---|---|---|---|
| **1. 开场定位** | 打开主界面，先指顶部标题与 mode 标签 | 这是飞腾多核弱网安全语义视觉回传系统，不是 generic benchmark | 15-20s |
| **2. current live 语义回传** | 选择稳定样本，固定 `SNR=10 dB`，点击“发送并重建” | 数据面真的跑在飞腾 Linux 侧；上位机做语义编码与弱网扰动，飞腾侧做重建 | 25-35s |
| **3. OpenAMP 控制面 live** | 点击“查询控制面状态”，必要时补 `JOB_ACK` / `SAFE_STOP` 证据卡 | 控制面真实存在，负责 admission、heartbeat、safe stop、fault handling | 20-30s |
| **4. performance 真值卡收口** | 展示 `4-core Linux performance mode` 指标卡 | headline performance 来自 4-core Linux mode，不能与 3-core demo mode 混写 | 15-20s |

这套顺序的核心是：**先 live 系统，再讲证据边界；先 current live，再讲 performance card；先控制面存在，再讲它不负责加速。**

### 3.2 推荐控件与默认设置

| 控件 / 区块 | 今晚是否主打 | 作用 |
|---|---|---|
| `探测板卡` | 是 | 证明飞腾板在线、系统身份可信、live 环境存在 |
| `发送并重建` | 是 | 主舞台动作，展示弱网语义视觉回传的数据面 |
| `查询控制面状态` | 是 | 展示 OpenAMP control plane 已在板级打通 |
| `4-core performance 真值卡` | 是 | 在不混 mode 的前提下给出 headline performance |
| `FIT 证据卡` | 备选 | 若现场不适合做 fault live，可展示已收口证据 |
| `推理引擎切换` | 否 | 可保留在附录或二级面板，不作为主叙事 |
| `批量测试` | 否 | 不是今晚上台主线，避免把故事讲回 benchmark |

推荐默认设置：

1. 主路径固定为 `current live`，不要把 baseline live 放在第一幕。
2. 主样本优先使用稳定图，`SNR` 默认固定在 `10 dB`，避免现场连续拖拽造成节奏分散。
3. 如果控制面 live 已稳定，可展示 `STATUS_REQ/RESP` 与 `JOB_REQ/JOB_ACK`；如果 fault live 风险较高，则切到 FIT 证据卡，不强行现场冒险。

def transfer_and_decode(noisy_latent, engine="tvm", use_pipeline=True):
    """SCP 传输到飞腾派 + SSH 触发远端推理 + SCP 回传结果"""
    tmp_dir = tempfile.mkdtemp(prefix="demo_")
    local_input = os.path.join(tmp_dir, "input.npy")
    local_output = os.path.join(tmp_dir, "output.png")
    remote_input = f"{REMOTE_WORK_DIR}/demo_input.npy"
    remote_output = f"{REMOTE_WORK_DIR}/demo_output.png"

    # 保存并传输
    np.save(local_input, noisy_latent)
    t_transfer_start = time.perf_counter()
    scp_to_pi(local_input, remote_input)
    transfer_up_ms = (time.perf_counter() - t_transfer_start) * 1000

    # 远端推理
    if engine == "tvm":
        decode_script = "demo_tvm_decode.py"
        pipeline_flag = " --pipeline biglittle" if use_pipeline else ""
    else:
        decode_script = "demo_mnn_decode.py"
        pipeline_flag = ""

    remote_cmd = (
        f"cd {REMOTE_WORK_DIR} && "
        f"{REMOTE_TVM_PYTHON} {decode_script} "
        f"--input {remote_input} --output {remote_output}"
        f"{pipeline_flag}"
    )
    rc, stdout, stderr = ssh_run(remote_cmd)

    # 解析远端输出的推理时间
    decode_ms = 0.0
    for line in stdout.strip().split("\n"):
        if "decode_ms=" in line:
            decode_ms = float(line.split("decode_ms=")[1].split()[0])

    # 回传结果
    t_transfer_back = time.perf_counter()
    scp_from_pi(remote_output, local_output)
    transfer_down_ms = (time.perf_counter() - t_transfer_back) * 1000

    transfer_ms = transfer_up_ms + transfer_down_ms

    if os.path.isfile(local_output):
        reconstructed = np.array(Image.open(local_output).convert("RGB"))
    else:
        reconstructed = np.zeros((256, 256, 3), dtype=np.uint8)

    return reconstructed, transfer_ms, decode_ms


# ────────────────────────────────────────
# 图像质量指标
# ────────────────────────────────────────

def compute_metrics(original, reconstructed):
    """计算 PSNR 和 SSIM"""
    from skimage.metrics import peak_signal_noise_ratio, structural_similarity

    orig = original.astype(np.float64) / 255.0
    recon = reconstructed.astype(np.float64) / 255.0

    h = min(orig.shape[0], recon.shape[0])
    w = min(orig.shape[1], recon.shape[1])
    orig = orig[:h, :w, :]
    recon = recon[:h, :w, :]

    psnr_val = peak_signal_noise_ratio(orig, recon, data_range=1.0)
    ssim_val = structural_similarity(orig, recon, data_range=1.0, channel_axis=2)
    return psnr_val, ssim_val

    fig, ax = plt.subplots(figsize=(5, 3))
    ax.hist(history, bins=20, color="#4A90D9", edgecolor="white", alpha=0.85)
    ax.axvline(np.median(history), color="#E74C3C", linestyle="--",
               linewidth=2, label=f"中位: {np.median(history):.1f} ms")
    ax.set_xlabel("解码延迟 (ms)", fontsize=10)
    ax.set_ylabel("次数", fontsize=10)
    ax.legend(fontsize=9)
    ax.set_title("推理延迟分布", fontsize=11, fontweight="bold")
    plt.tight_layout()
    return fig


def make_comparison_chart():
    """生成 4-core Linux performance mode 真值卡柱状图"""
    engines = ["PyTorch\nreference", "TVM\nserial", "TVM\nbig.LITTLE"]
    latencies = [484.183, 231.522, 134.617]

    plt.tight_layout()
    return fig


# ────────────────────────────────────────
# 主流程
# ────────────────────────────────────────

def run_single(image, snr, engine, use_pipeline):
    """单张图片：编码 → 传输 → 解码 → 质量评估"""
    global latency_history

    if image is None:
        return (None, "请先选择或上传一张图片", "", "",
                make_latency_chart([]), make_comparison_chart())

    try:
        # 编码
        noisy_latent, encode_ms = encode_image(image, snr)

        # 传输 + 解码
        reconstructed, transfer_ms, decode_ms = transfer_and_decode(
            noisy_latent, engine, use_pipeline
        )
        total_ms = encode_ms + transfer_ms + decode_ms

        # 质量评估
        psnr_val, ssim_val = compute_metrics(image, reconstructed)

        # 记录延迟
        latency_history.append(decode_ms)

        # 格式化输出
        timing_text = (
            f"编码耗时:    {encode_ms:>8.1f} ms\n"
            f"传输耗时:    {transfer_ms:>8.1f} ms\n"
            f"解码耗时:    {decode_ms:>8.1f} ms  [{engine.upper()}]\n"
            f"{'─' * 36}\n"
            f"端到端总计:  {total_ms:>8.1f} ms\n"
            "说明: headline performance 请看独立的 4-core mode 真值卡"
        )
        quality_text = f"PSNR: {psnr_val:.2f} dB    SSIM: {ssim_val:.4f}"
        config_text = (
            f"推理引擎: {engine.upper()}"
            + (" + big.LITTLE 流水线" if use_pipeline else "")
            + f"\nSNR: {snr} dB"
        )

        return (reconstructed, timing_text, quality_text, config_text,
                make_latency_chart(latency_history), make_comparison_chart())

    except Exception as e:
        return (None, f"错误: {e}", "", "",
                make_latency_chart(latency_history), make_comparison_chart())


def run_batch(num_images, engine, use_pipeline, progress=gr.Progress()):
    """批量模式：连续发送多张图片"""
    global latency_history
    latency_history = []

    gallery_images = sorted(glob.glob(os.path.join(GALLERY_DIR, "*.png")))
    gallery_images += sorted(glob.glob(os.path.join(GALLERY_DIR, "*.jpg")))

    if not gallery_images:
        return "错误: 未找到图库图片", make_latency_chart([])

    results = []
    for i in progress.tqdm(range(min(num_images, len(gallery_images))),
                           desc="批量推理中"):
        img = np.array(Image.open(gallery_images[i]).convert("RGB"))
        noisy_latent, _ = encode_image(img, snr=10)
        _, _, decode_ms = transfer_and_decode(noisy_latent, engine, use_pipeline)
        latency_history.append(decode_ms)
        results.append(decode_ms)

    summary = (
        f"批量推理完成\n"
        f"{'─' * 36}\n"
        f"图片数量:   {len(results)}\n"
        f"中位延迟:   {np.median(results):.1f} ms\n"
        f"平均延迟:   {np.mean(results):.1f} ms\n"
        f"最小延迟:   {np.min(results):.1f} ms\n"
        f"最大延迟:   {np.max(results):.1f} ms\n"
        f"总耗时:     {sum(results) / 1000:.1f} s\n"
        f"吞吐量:     {len(results) / (sum(results) / 1000):.1f} 张/秒"
    )
    return summary, make_latency_chart(latency_history)

### 4.3 飞腾派端解码脚本

需要在飞腾派上部署一个轻量解码脚本，供 Demo 的 SSH 调用。

```python
#!/usr/bin/env python3
"""demo_tvm_decode.py — 飞腾派端 TVM 解码脚本（供 Demo SSH 调用）"""

import argparse
import time

import numpy as np

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--input", required=True, help="输入 .npy 文件路径")
    parser.add_argument("--output", required=True, help="输出 .png 文件路径")
    parser.add_argument("--artifact", default="tvm_tune_logs/optimized_model.so")
    parser.add_argument("--pipeline", default="none",
                        choices=["none", "biglittle"])
    args = parser.parse_args()

    import tvm
    from tvm import relax
    from PIL import Image

    # 加载模型
    dev = tvm.cpu(0)
    lib = tvm.runtime.load_module(args.artifact)
    vm = relax.VirtualMachine(lib, dev)

    # 加载输入
    latent = np.load(args.input).astype(np.float32)
    if latent.ndim == 3:
        latent = np.expand_dims(latent, 0)
    tvm_input = tvm.nd.array(latent, dev)

    # 推理计时
    vm["main"](tvm_input)  # warmup
    t0 = time.perf_counter()
    output = vm["main"](tvm_input)
    decode_ms = (time.perf_counter() - t0) * 1000

    # 保存重建图像
    output_np = output.numpy()[0]  # [3, H, W]
    img_np = output_np.transpose(1, 2, 0)  # → [H, W, 3]
    img_np = np.clip(img_np * 255, 0, 255).astype(np.uint8)
    Image.fromarray(img_np).save(args.output)

    # 输出指标（供上位机解析）
    print(f"decode_ms={decode_ms:.3f}")
    print(f"output_shape={list(output_np.shape)}")

if __name__ == "__main__":
    main()
```

### 5.2 录制注意事项

**画面：**
- 使用 OBS Studio 录屏，分辨率 1920×1080
- 如果有条件，外接一个摄像头对准飞腾派实物，画中画嵌入
- 界面字体确保在 1080p 下清晰可读，顶部 mode 标签和 control-plane 状态条必须清楚
- 主画面优先给 current live 重建区和控制面状态区，不要把镜头时间浪费在次要按钮上

**音频：**
- 提前写好画外音稿，录制时对照朗读
- 语速适中，关键数字放慢强调
- 背景安静，使用耳机麦克风

**节奏：**
- 每个镜头之间自然过渡，不要硬切
- 点击“发送并重建”或“查询控制面状态”后的等待时间，用口播解释 data plane / control plane 分工
- performance 指标建议作为单独一页卡片出现，不要和 demo mode live 镜头交叉剪辑成同一种口径

**预演：**
- 正式录制前至少预演 2 遍
- 确保飞腾派 SSH 稳定、不会中途断连
- 预先跑通 current live 路径和 control-plane 状态查询路径，避免录制时出错
- 准备 1-2 张效果稳定的图片作为主样本，避免录屏当天临时换图
- 如果 fault live 演示不够稳定，提前准备 FIT 证据页，视频里可以切证据而不是硬做 live 注入

### 5.3 今晚上台不建议主打的内容

- 不把 baseline live 当作主舞台路径。baseline 可以作为附录或证据，不要抢 current live 的主叙事。
- 不把 `TVM / MNN` 引擎切换当作核心卖点。今晚的核心是弱网语义回传系统和 OpenAMP control plane，不是多框架秀。
- 不把批量 50 张或 300 张吞吐测试作为主线。吞吐结果可以保留在 performance evidence 区，但不建议现场主打。
- 不把 `4-core Linux performance mode` 数字和 `3-core Linux + RTOS demo mode` live 画面混剪成一个结论。
- 不说“OpenAMP 让推理更快”。当前应诚实表述为：OpenAMP 提供 control plane / safety / admission / heartbeat / safe stop。
