## 2. 口径冻结规则

- 对外主画面、Demo、答辩、总述类材料，只允许引用本文件中的 trusted current SHA 与两条正式 benchmark 数值。
- 严禁把 `payload-only` 与 `real reconstruction` 混写在同一条结论中。
- `130.219 ms` 只能表示 payload 口径，不得改写成真实端到端。
- `230.339 ms/image` 只能表示 read latent -> reconstruct -> write PNGs 的真实端到端口径，不得与 Scheme B、payload runner 结果混写。
- 历史 SHA（如 `65747fb3...b6377`、`1946b08e...c644`）和历史数值继续保留，但仅作为版本谱系或里程碑证据，不再作为“当前 trusted current”引用。

## 4. 硬件与 target 表述策略

### 对外平台表述

- 平台/作品主叙事使用飞腾平台官方表述，强调“基于飞腾平台的安全可靠图像语义通信系统”。
- OpenAMP 作为异构监护控制面能力描述，不把其写成 TVM 推理数据面替代实现。

### TVM 优化页表述

- TVM 优化页单独说明当前稳定 target 为 `llvm / aarch64-linux-gnu / cortex-a72 + neon / num-cores=4`。
- `+crypto,+crc` 只作为受控实验 target，不进入当前正式默认口径。
- baseline 与 current runtime 路径必须分开说明：
  - baseline：兼容 legacy/compat 路径
  - current：`tvm310_safe + safe 0.24.dev0 runtime`
