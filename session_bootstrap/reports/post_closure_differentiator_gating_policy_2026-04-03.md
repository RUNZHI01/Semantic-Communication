# P2 差异化后置 / 暂缓 / 备线策略说明（2026-04-03）

## 目的

把总清单里这些长期容易混在一起的条目拆开：

- 哪些是**后置差异化项**
- 哪些是**明确暂缓项**
- 哪些是**保留但不抢主叙事的备线**

这样后续就不会继续把“现在不做”误解成“没人管”，也不会把“以后可做”误解成“当前主线缺口”。

---

## 1. 当前总原则

在 2026-04-03 这个时间点，项目优先级已经正式收敛为：

1. 先补真实闭环（Demo 彩排 / OpenAMP 真机协议剩余项 / judge-facing 实测扩样本）
2. 再做国奖差异化扩展

因此，下面这些策略现在应视为**已经建立的执行规则**，而不是继续挂在清单里的未定事项。

---

## 2. 保留但不抢主叙事的备线

### 2.1 MNN 的正确定位

MNN 当前应固定为：

- 动态尺寸支持备线
- 量化探索备线
- 对比路线 / 补充路线

而不应抢当前主叙事。

当前主叙事仍然是：

- TVM trusted current：固定形状高性能解码主线
- OpenAMP：控制面 / 安全边界 / 异构监护

因此，`将 MNN 保留为动态尺寸与量化备线，不抢主叙事` 这条本质上已经属于**定位决策完成**。

---

## 3. 明确后置的差异化项

这些项不是取消，而是必须排在 P0/P1 真闭环之后：

- big.LITTLE 异构流水线进一步纳入 Demo / 答辩
- 跨框架对比：TVM vs MNN vs NCNN / ORT
- TVM hotspot 定向深搜
- TVM 量化探索
- 手写 TIR 等更硬核方向

### 当前判断

这些方向都可能有价值，但在当前阶段：

- 不能压过 Demo 真实彩排
- 不能压过 OpenAMP 剩余真机协议证据
- 不能压过 judge-facing 现有主证据的收口

因此，`TVM 量化只作为试探性任务，必须排在 OpenAMP、FIT、Demo、质量验证之后` 这条，也已经属于**优先级规则建立完成**。

---

## 4. 当前明确暂缓项

以下事项当前应明确标记为“已决定暂缓”，而不是“还没决定要不要做”：

- 自定义 Pass
- 动态 TVM
- 模型剪枝重训
- 把 OpenAMP 做成大数据面重写工程
- 把“继续压几毫秒 payload”放在赛题闭环之前

### 为什么这些可以视为“策略已完成”

因为当前不是缺少判断，而是判断已经形成：

- **自定义 Pass / 动态 TVM / 剪枝重训**：都属于高投入、偏后续阶段的差异化方向
- **OpenAMP 大数据面重写**：与“OpenAMP 只做控制面、不重写 TVM current 数据面”原则冲突
- **继续压几毫秒 payload 优先于赛题闭环**：与当前总优先级冲突

也就是说，这些条目当前更准确的状态不是“未完成”，而是：

> **已明确暂缓 / 已列入后续工作**

---

## 5. 对任务板的建议解释

因此，总清单中的这些条目应拆成两类：

### A. 已建立策略，可标完成

- MNN 保留为动态尺寸与量化备线，不抢主叙事
- TVM 量化只作为试探性任务，排在 OpenAMP/FIT/Demo/质量验证之后
- 暂缓自定义 Pass
- 暂缓动态 TVM
- 暂缓模型剪枝重训
- 暂缓把 OpenAMP 做成大数据面重写工程
- 暂缓把“继续压几毫秒 payload”放在赛题闭环之前

### B. 真正仍未完成的未来扩展项

- 正式完成 big.LITTLE 异构流水线数据并纳入 Demo / 答辩
- 补跨框架对比
- 继续做 TVM hotspot 定向深搜
- 如果时间充裕，再做手写 TIR 等硬核方向

---

## 6. 一句话结论

> 当前项目不是“什么都还没做”，而是已经明确区分了：什么是当前主线闭环、什么是保留备线、什么是后置差异化、什么是明确暂缓。后者这些策略项现在可以视为决策完成，而不必继续挂成未定状态。
