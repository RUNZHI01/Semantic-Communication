# 布局彻底重新设计 - 大字体、大控件、大占比

**日期**: 2026-03-30
**目标**: 解决"看不清"的问题，左右两栏占据主要视觉空间

---

## 设计决策

### 1. 布局架构

**保留Hero但缩小**
- Hero区域限制在 25vh 高度（之前是 flex: 1）
- 主要内容区域（左右两栏）占据剩余 75% 高度

**左右两栏等宽**
- 从 480px + 自适应 → **1fr + 1fr**（各占50%）
- 左右两栏成为视觉主体，占据整个画面宽度

### 2. 字体大小全面升级

| 元素 | 旧大小 | 新大小 | 增幅 |
|------|--------|--------|------|
| **面板标题** | 14px | **19px** | +36% |
| **正文文字** | 13-15px | **18-20px** | +33% |
| **标签文字** | 11-13px | **15-17px** | +31% |
| **按钮文字** | 14-16px | **17-19px** | +26% |
| **Hero数字** | 72px | **64px** | -11% (适当缩小) |
| **Hero标签** | 14px | **15px** | +7% |
| **次要指标** | 16px | **20px** | +25% |

### 3. 按钮和控件尺寸

| 按钮类型 | 旧尺寸 | 新尺寸 | 增幅 |
|----------|--------|--------|------|
| **主按钮** | 18×28px padding | **22×32px padding, 58px高度** | +53% |
| **次按钮** | 16×24px padding | **20×28px padding, 54px高度** | +50% |
| **危险按钮** | 14×20px padding | **18×24px padding, 52px高度** | +43% |
| **恢复按钮** | 16×24px padding | **20×28px padding, 54px高度** | +50% |
| **状态按钮** | 14×20px padding | **18×24px padding, 52px高度** | +43% |

**按钮高度**: 52-58px (之前是 40-48px)
**按钮padding**: 18-24px (之前是 12-18px)
**图标间距**: 12-14px (之前是 8-12px)

### 4. 间距优化

| 区域 | 旧间距 | 新间距 | 变化 |
|------|--------|--------|------|
| **面板gap** | 24px | **28px** | +17% |
| **状态项padding** | 16px | **20px** | +25% |
| **section gap** | 16px | **保持** | - |
| **内边距** | 32px | **24-32px** | -25% |

## 视觉占比变化

### 之前（小侧栏）
```
┌─────────────────────────────────────┐
│        Hero区域 (flex: 1)           │ ← 占据40%高度
├─────────────────────────────────────┤
│ [480px] │      自适应中间区域        │ ← 左侧窄
│         │                           │
│         │                           │
└─────────┴───────────────────────────┘
```

**问题**: 左侧太窄，字体小，看不清

### 现在（两栏主导）
```
┌─────────────────────────────────────┐
│   Hero区域 (max-height: 25vh)       │ ← 压缩到25%
├──────────────────────┬──────────────┤
│                        │              │
│   左侧面板 (50%)        │ 右侧面板(50%)  │ ← 各占一半
│                        │              │
│   系统状态+数据         │   操作区域     │
│                        │              │
└────────────────────────┴──────────────┘
```

**优势**:
- ✅ 左右两栏成为视觉主体
- ✅ 信息密度高但清晰
- ✅ 字体大、控件大
- ✅ 易读、易操作

## 响应式设计

### 大屏幕 (>2000px)
```css
.mainContent {
  grid-template-columns: 1.2fr 0.8fr;  /* 左侧稍大 */
  gap: 32px;
}
```

### 标准屏幕 (1200-2000px)
```css
.mainContent {
  grid-template-columns: 1fr 1fr;  /* 等宽 */
  gap: 28px;
}
```

### 中等屏幕 (900-1200px)
```css
.mainContent {
  grid-template-columns: 1fr 1fr;  /* 保持等宽 */
  gap: 24px;
  padding: 20px 24px;
}
```

### 小屏幕 (<900px)
```css
.mainContent {
  grid-template-columns: 1fr;  /* 堆叠 */
  gap: 20px;
  padding: 16px 20px;
}
```

## 设计原则

### 1. 功能优先
- 不是"好看"，而是"好用"
- 不是"极简"，而是"清晰"
- 不是"艺术"，而是"实用"

### 2. 信息密度
- 高密度但不拥挤
- 多信息但清晰
- 大字体但不浪费空间

### 3. 交互友好
- 按钮大而易点
- 文字大而易读
- 布局熟悉而直观

### 4. 专业可信
- Apple级别的精致感
- 仪表盘级别的信息密度
- 演示级别的视觉冲击
